# 许白集团在职证明工具 — 开发记录与功能清单

> 本文档记录"在职证明工具-离线版.html"从需求到当前状态的完整演变,以及当前所有功能与校验规则,供存档与交接使用。
> 最后更新:对话进行中的最新状态。

---

## 一、工具概述

一个**纯前端、零依赖、完全离线**的网页工具,用于填写并生成"许白集团成员企业 员工履历(在职)证明"Word 文档(.docx)。

核心特性:
- 单个 HTML 文件,双击用浏览器打开即可使用,无需安装、无需联网、不上传服务器。
- 左侧表单填写,右侧 A4 版式实时预览,所见即所得。
- 点击导出按钮生成 .docx,用 Word / WPS 均可打开。
- 生成 Word 的逻辑完全用原生 JavaScript 手写(手写 zip 打包 + docx XML),不引用任何 CDN 或外部库,因此断网也能用。

---

## 二、完整开发记录(按时间顺序)

下面是这个工具每一次迭代的需求与改动。

### 1. 起点:通用 Word 查找替换工具
最初做的是一个纯前端的 Word 文档查找替换工具(拖入 .docx、批量查找替换、保留格式)。随后需求转向针对某一份具体模板的定制工具。

### 2. 针对"在职证明"模板的填写预览工具
根据上传的"许白集团成员企业在职证明模板.docx",做了一个定制工具:左侧表单、右侧实时预览、可导出 Word。当时依赖 CDN 加载 docx 库。

### 3. 加入登录功能(后被移除)
- 曾尝试加入真实账号系统:FastAPI 后端 + JWT + bcrypt + 登录限速,前端登录页 + 受保护页面。
- 提供了部署方案(阿里云 ECS / 云服务器 + nginx + https + systemd)。
- **该登录功能后续按要求移除**,回到纯前端单文件形态。配套的后端代码(cert-app 文件夹)单独保留,未并入主工具。

### 4. 改为"真正能生成文件"的离线版(关键转折)
- 原版点下载没反应,根因是从 CDN 加载库失败(断网或网络拦截)。
- 改为**零依赖**:.docx 本质是 zip + XML,改用纯 JS 手写 zip(store 模式)和文档 XML,不引用任何外部库。
- 经验证:生成的 docx 通过 zip 完整性校验,并用 LibreOffice 实际打开确认可正常渲染。

### 5. 排版对齐预览
调整导出 Word 的行距、字号、表格疏密、留白,使其观感与网页预览一致(网页按像素、Word 按 A4,做不到逐像素一致,但风格统一)。

### 6. 身份证明文件号码 + 证件类型校验
- "身份证号码"字段改为"身份证明文件号码"。
- 加入证件类型下拉,按类型校验位数,不通过则禁止生成。
- 居民身份证含末位校验码(国标算法)校验。

### 7. 入职/离职时间改为日期选择器
- "入职时间"改为日期选择器,显示为 `YYYY/MM/DD`。
- "离职时间"加"在职/已离职"开关:在职显示 `N/A`,已离职才出现日期选择器并显示具体日期。

### 8. 有效期至日期选择器
- "有效期至"改为日期选择器;留空时自动取"申请日期 + 6 个月"(含月末溢出、闰年、跨年的边界处理)。

### 9. 申请时间字段 + 正文改写
- 正文那段改为:"您好！您在 (申请时间) 于许白集团员工人事系统(people.xubainet.cn)申请了关于您本人在我公司的履历……","我司"改为"我公司"。
- 新增"申请时间"日期选择器,留空则默认用系统当天日期。

### 10. 工资字段拆分为货币 + 数额
- "工资"拆成货币下拉 + 数额输入。
- 数额留空显示"N/A(以实际为准)";填写显示"货币 + 数额"。

### 11. 货币符号规则,移除 EUR
- 货币去掉 EUR,只保留 CNY / HKD / USD。
- 显示符号:USD → `US$`、HKD → `HK$`、CNY → `CN￥`(全角￥)。

### 12. 姓名与证明对象数据互通 + 姓名校验
- "证明对象(员工)"做成"姓名"的只读镜像,自动同步。
- 姓名校验:仅限纯中文或纯英文、不可混合;中文≥2 字,英文≥2 字母。
- 英文允许空格、连字符、撇号(但需至少 2 个真正字母);中文允许间隔号·。

### 13. 岗位类型改为下拉(含"其他")
- "岗位类型"改为下拉:全职 / 实习 / 兼职 / 其他。
- 选"其他"时出现输入框,必填具体内容。

### 14. 身份证出生日期合法性校验
- 居民身份证第 7–14 位(YYYYMMDD)必须是真实存在的日期,否则不通过。
- 例:530112201012136614(20101213 合法)通过;530112201020206614(20 月非法)不通过。
- 含闰年 2/29、月末溢出等边界处理。

### 15. 新增/拆分证件类型
- 新增"外国人永久居留身份证"。
- 原"港澳台居民居住证"拆为"港澳居民居住证"和"台湾居民居住证"。
- 这三类与居民身份证**校验规则相同**(18 位 + 出生日期合法 + 末位校验码)。

### 16. 证件类型显示名加国名前缀
- "居民身份证" → "中华人民共和国居民身份证"。
- "外国人永久居留身份证" → "中华人民共和国外国人永久居留身份证"。
- (港澳/台湾居住证此前已是"中华人民共和国……居住证"。)

### 17. 负责人字段默认值
- 负责人或上级:留空显示 `N/A`。
- 负责人电邮:留空显示 `N/A`。
- 负责人电话:留空显示 `+86-400-991-5178`。
- 负责人传真/座机:留空显示 `N/A`(此项稍后补充)。

### 18. 入职/离职日期先后校验
- 选了离职日期时,离职时间必须**晚于**入职时间(严格大于),否则不通过。
- 未选离职(在职)或缺任一日期则不校验。
- 例:入职 20250823 / 离职 20250921 通过;入职 20250821 / 离职 20230521 不通过。同一天判不通过。

### 19. 工资数额必须为阿拉伯数字
- 填写时只接受阿拉伯数字(允许千分位逗号与小数点)。
- "一二三"、罗马数字"I"、带单位、字母等一律不通过。留空不校验。

### 20. 工资首位不能为 0
- 首位不能是 0(如 001832.22 不通过,1832.22 通过)。
- 例外:单个"0"、以及"0.xx"这类小于 1 的纯小数仍允许。

### 21. 负责人传真/座机默认值
- 留空显示 `N/A`(补齐第 17 项中的传真项)。

### 22. 生成文档追加证件影印件页
- 生成时可在正文后追加一页"身份证明文件影印件",放上传的图片。
- 图片居中、等比缩放到页面可用宽度内,纵向排列;Word 自动续页。
- 图片全程仅在本机处理、嵌入文档(手写 docx 嵌图:media 文件 + content-types + 关系 + 新页),不上传。
- 当时要求至少 1 张、最多 4 张。

### 23. 岗位类型与年龄匹配校验
- 由身份证第 7–14 位出生日期按**系统当日**算周岁,对照岗位最低年龄:
  - 全职:≥18 周岁
  - 实习:≥16 周岁
  - 兼职:≥12 周岁
  - 其他:不校验(填写人自行判断)
- 规则文字显示在岗位类型区域;不达标则提示并禁止生成。
- 仅对含标准出生日期的四类证件生效(居民身份证、港澳/台湾居住证、外国人永居证);护照等不参与。
- 年龄按周岁计(生日未到则减一岁),临界精确到月日。

### 24. 18 位证件输入分组显示
- 四类 18 位中国证件在**输入框**里实时显示为"六位地区码 + 空格 + 八位生日 + 空格 + 后四位"。
- 例:输入 530112201012136614 → 显示"530112 20101213 6614"。
- 校验、年龄判断、文档输出**全程用去空格的原始号码**(文档仍写 530112201012136614)。

### 25. 证件影印件改为非必填(当前最新)
- 上传图片改为可选:不上传则文档不含影印件页、也不拦截生成;上传了才显示影印件页。
- 仍保留最多 4 张上限。

| 证件类型(显示名) | 内部值 | 校验规则 | 参与年龄校验 |
|---|---|---|---|

| 其他证件 | other | 非空即可 | 否 |

### 关键校验汇总(不通过则禁止生成)
1. **姓名**:纯中文≥2 字 或 纯英文≥2 字母,不可混合。
2. **证件号**:按所选证件类型校验;四类 18 位证件还校验出生日期合法性与末位校验码。
3. **岗位"其他"**:必须填写具体内容。
4. **离职日期**:若已选,须晚于入职日期。
5. **工资数额**:若填写,须为阿拉伯数字、首位非 0(允许 0 与 0.xx)。
6. **年龄**:全职≥18 / 实习≥16 / 兼职≥12(基准系统当日,仅四类身份证件;其他岗位不校验)。
7. **影印件**:可选;若上传则最多 4 张。

### 文档生成行为
- 正文为单页 A4 版式:抬头、标语、标题、致辞段(含申请时间)、七行信息表、两段证明语(含证明对象、有效期下划线)、盖章位。
- 中文字体宋体;货币、N/A、日期等按上述规则呈现。
- 若上传了影印件,正文后追加一页"身份证明文件影印件",图片居中等比排列。
- 输出文件名:`<姓名>-在职证明.docx`。

## 四、技术与维护说明

- **零依赖**:不引用任何 CDN / npm 库;生成 Word 用纯 JS 手写 zip + docx XML。
- **隐私**:所有数据(含证件图片)仅在本机浏览器内处理,不联网、不上传。
- **验证方式**:每次改动均在 Node 环境跑校验逻辑用例,并用 LibreOffice 将生成的 .docx 渲染成图片肉眼确认版式。
- **已知小限制**:
  - 网页预览按屏幕像素、Word 按 A4 纸张,版式观感一致但非逐像素相同。
  - 18 位证件输入框自动分组时,若在号码中间增删字符,光标会跳到末尾(顺序输入/粘贴无影响)。
  - 四类身份证件统一沿用居民身份证校验逻辑(现实中各证件编码规则略有差异,按需求统一处理)。
  - 浏览器原生日期选择器外观因浏览器而异,但输出格式统一为 YYYY/MM/DD。
- **附:被移除的登录功能**:FastAPI + JWT + bcrypt 的后端方案(cert-app)单独保留,未并入主工具;如需带登录部署可单独启用。
